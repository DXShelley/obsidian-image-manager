# 更新日志

## 4.0.5

- 在插件 `manifest.json` 中新增 `fundingUrl`，指向项目支持页面，让 Obsidian 社区目录可展示支持入口。
- 新增网站“支持项目”区域，展示微信支付、微信赞赏码和支付宝二维码，并同步 README 与 GitHub Funding 配置。
- 将支持页面二维码替换为正方形 PNG 版本，便于在网站卡片中完整展示。

## 4.0.4

- 修复旧版 Obsidian 中 `ButtonComponent.setDestructive()` 不存在时设置页渲染中断的问题，重置按钮会自动回退到 `setWarning()` 或 `mod-warning` 样式。
- 补充设置页兼容性回归测试，并扩展测试用 Obsidian mock，使发布前验证覆盖旧版按钮 API 场景。
- 同步更新 `manifest.json`、`package.json`、`versions.json`、README、任务状态与网站版本文案，准备以 `4.0.4` tag 重新发布。

## 4.0.3

- 调整 `manifest.json` 描述，移除审核规则认为冗余的 `Obsidian` 字样。
- 将 `minAppVersion` 提升到 `1.13.1`，与当前使用的 Obsidian API 类型版本保持一致。
- 按社区插件审核反馈替换直接样式赋值、原生标题元素、`fetch`、永久删除、全局 `document`、已废弃设置控件 API，并在粘贴处理前尊重 `event.defaultPrevented`。
- 补充 README 与文档索引中的开源依赖、框架、工具链和参考项目致谢。

## 4.0.1

- 将社区目录展示名从泛用的 `Image Manager` 调整为更唯一的 `Note Image Manager`，降低与既有社区插件重名导致的审核退回风险。
- 发布工作流新增安装 zip 附件，方便用户直接下载完整安装包。
- 设置页顶部标题改用 Obsidian 推荐的 `Setting.setHeading()` 渲染，减少人工审核中的 UI 规范评论。

## 4.0.0

- 新增集中式双语文案体系，设置页、功能状态、命令名、通知、右键菜单与风险确认会按界面语言输出。
- 语言切换后自动刷新已注册命令，保持范围命令排序、命令面板文案与设置页语言一致。
- 强化外部图片导入：阅读视图可右键只导入当前单张外部图片，粘贴文本源支持更完整的失败提示，显式导入可根据响应 `content-type` 识别无后缀或动态图片 URL。
- 新增转换忽略正则、转换跳过提示和批量转换总结；`AVIF` 纳入图片识别、外链导入与转换输入链路，同时保留原位编辑 / 压缩前需先转为兼容格式的约束。
- 压缩历史按当前文件版本持久化，避免重复压缩同一版本或重复尝试无收益压缩。
- 统一图片链接解析、预览刷新、画廊灯箱关闭行为和操作反馈，覆盖中文编码路径混用、预览缓存刷新与失败通知等边界。
- 补充 i18n、主插件命令刷新、预览外链导入、压缩、转换、文件管理和反馈文案相关单元测试，并同步更新 README、用户指南、任务状态与测试用例。

## 3.1.0

- 将去水印调整为规划能力，只在功能状态与网站中保留说明，不再暴露未达标的处理实现。
- 设置页与功能状态支持中英文切换，默认使用简体中文，并统一收敛配置项文案与说明。
- 重写项目网站的中英文语义、字体和信息层级，补充“够用实用、不制造熵”的项目理念。
- 更新 README、用户指南、任务状态、测试用例与致谢，并补充对 `obsidian-image-converter` 的感谢。

## 3.0.0

- 新增右键“拖拽裁剪”图片处理流程，并统一纳入恢复事务，确保修改后可撤销 / 重做。
- 修复阅读视图双击图片打开单图画廊、在来源笔记范围内预选当前图片，以及仅清理图片附件目录空文件夹等画廊与目录整理细节。
- 重新接回当前图片旋转、翻转与 1920px 边界缩放命令的功能注册，修正功能状态、设置开关与实际可用能力之间的不一致。
- 补充裁剪、功能目录和开关可见性相关单元测试，并同步清理过期文档与测试说明。
- 新增独立 `website/` 宣传主页，用于展示当前能力、功能状态和规划路线。

## 2.0.0

- 将当前笔记范围的转换与压缩命令改为批量处理该笔记引用的全部图片，并延迟合并 Markdown 预览刷新。
- 调整命令面板中的范围型命令排序，统一按 `【单文件】`、`【单文件夹】`、`【整库】` 分组展示，仅在 `id` 中保留 `a/b/c/d` 排序前缀。
- 为所有整库命令增加执行前风险确认，降低误操作风险。
- 扩展恢复系统为可撤销 / 重做最近事务，并为事务记录补充 before / after 状态快照。
- 新增压缩历史跟踪，避免对同一文件版本重复压缩，且在无收益压缩时给出明确跳过提示。
- 新增“删除空图片文件夹”“删除孤立图片”控制项，以及当前笔记 / 当前文件夹 / 整库三个范围的 `删除多余图片文件` 命令。
- 孤立图片清理在检测到外部唯一引用时，会自动迁移到对应笔记的受管目录；存在多个外部引用时则保留文件。
- Markdown 图片链接新增编码、可读包裹和自动三种路径输出策略，兼容中文、空格、括号以及已编码路径混用。
- 画廊增强为支持当前图片入口、阅读视图双击打开、右键菜单打开，以及在来源笔记范围内预选当前图片。
- 新增中英文文档索引、用户指南、架构说明、测试用例和更新日志补充。

## 1.0.0

- `Note Image Manager` 首个稳定版本。
- 将插件按 `app`、`core`、`features`、`services`、`types`、`ui` 与 `utils` 模块化拆分。
- 增加可配置的粘贴处理、保存路径模板、重命名模板与实时设置预览。
- 为适用功能增加当前文件、当前文件夹与整个仓库范围的命令。
- 增加按受管目录迁移图片并批量更新图片链接的能力。
- 增加图片压缩、格式转换、缩放预设、旋转与翻转命令。
- 增加笔记与文件夹画廊视图，支持排序、筛选与网格大小切换。
- 增加右键菜单集成、图片替换后的预览刷新，以及 CI / 发布工作流。
