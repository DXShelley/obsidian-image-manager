# 重构状态

## 发布

- `v3.1.1` 将社区首次提交前的 README / Release / Pages 对齐、外部图片单图导入入口收口，以及无 `v` 前缀的正式发版路径纳入正式发布范围。
- 包元数据、清单元数据、发布工作流、宣传主页与面向用户的文档已经对齐到 3.1.1 发布状态。

## 已完成

- 核心设置管理器、事件总线与功能注册中心
- 基于功能模块的命令和监听器注册
- 将设置 UI 与模态框 UI 拆分到 `src/ui`
- CI 与发布工作流脚手架
- 更新 README 与架构文档
- 为不暴露 `Setting.setErrorMessage()` 的 Obsidian 版本提供设置兼容兜底
- 将保存路径与生成文件名预览移动到设置页顶部
- 当 `{noteName}-{date}` 等相邻变量解析为相同值时自动折叠重复片段
- 原位旋转、翻转、缩放、压缩或转换图片后自动刷新 Markdown 预览
- 笔记与文件夹画廊模态框，支持筛选、排序、网格 / 列表模式与灯箱预览
- 面向当前笔记、当前文件夹与整个仓库的批量图片链接重写，并支持将图片迁移至配置目录
- 面向当前文件、当前文件夹与整个仓库的图片压缩
- 面向当前文件、当前文件夹与整个仓库的图片格式转换
- 提供可单独启用或关闭的文件右键菜单操作
- 提供可单独启用或关闭的操作通知
- 压缩与转换忽略正则设置，带行内校验与示例
- 设置页与功能状态支持中英文切换，默认语言为简体中文
- 默认渲染图片对齐，以及可选的点击时抑制 Obsidian 图片选中行为
- 自动重命名开关，在关闭时保留原始文件名
- 支持将粘贴的图片文本来源自动下载为文件，包括图片 URL、`file://` 路径与 `data:image/...;base64,...`
- 支持通过命令面板按当前文件、当前文件夹、整个仓库范围下载外部图片到本地
- 支持在阅读视图中右键单张外部图片，只导入当前选中的这一张
- 为图片重写、路径移动、笔记链接重写与粘贴 / 导入流程持久化恢复事务

## 部分完成

- 图片编辑器：已交付快速旋转 / 翻转操作，但交互式画布编辑 UI 仍待实现
- 缩放：已交付预设缩放命令，但高级预设与 UI 仍待实现

## 近期集成说明

- 如果设置页在 Obsidian 中显示不完整，先确认仓库中的插件目录已复制最新的 `main.js`、`manifest.json` 与 `styles.css`。
- `outputFolder` 仅影响插件接管粘贴流程时保存的图片；如果 `enablePasteHandler` 关闭，则保存位置由 Obsidian 或其他插件决定。
- 旋转与翻转会立即重写底层图片文件；Markdown 预览刷新依赖当前笔记仍然引用该图片路径。
- 忽略正则设置以 `file.path` 为匹配对象，支持每行一个正则，并允许以 `#` 开头的注释行。
- Markdown 图片路径解析已统一为“原始路径优先，解码路径兜底”，用于覆盖同一笔记内混合的 `%E6...` 编码中文路径与未编码中文路径。
